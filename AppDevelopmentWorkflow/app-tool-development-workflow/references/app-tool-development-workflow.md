# 應用程式 / 工具開發完整流程

> 適用於個人到小隊的軟體開發：小型 Web App、內部工具、CLI、自動化腳本、Dashboard、API 服務、資料處理工具與 AI 協作開發。

## 1. 需求釐清

目標是把模糊想法變成可判斷、可取捨、可驗收的專案方向。

### 必要問題

| 項目 | 說明 |
|---|---|
| 問題 | 要解決什麼具體痛點？ |
| 使用者 | 誰會使用？使用頻率與環境是什麼？ |
| 現況 | 現在如何處理？缺點是什麼？ |
| 成功標準 | 完成後如何判斷有用？ |
| 範圍 | 這次要做什麼、不做什麼？ |
| 限制 | 平台、時間、資料來源、權限、部署、成本限制是什麼？ |

### 產出

- `project-brief.md`
- 一句話目標
- 主要使用情境
- 成功條件
- In scope / Out of scope
- 待確認事項

## 2. 規格設計

目標是把需求轉成工程師或 Codex 可以實作的規格。

### 內容

- 功能需求：使用者能完成哪些動作。
- 非功能需求：效能、安全、穩定性、相容性、可維護性。
- 輸入與輸出：來源、格式、限制、錯誤情境。
- 主要流程：正常流程、例外流程、空狀態。
- 權限與資料：登入、授權、敏感資料、保存與刪除。
- 驗收條件：每個重要功能都要能測試。

### 產出

- `system-spec.md`
- `FR-001` 類功能需求
- `NFR-001` 類非功能需求
- `AC-001` 類驗收條件
- Mermaid 流程圖或序列圖，視需要加入

## 3. 技術設計

目標是決定如何實作，並留下重要取捨。

### 內容

- 應用型態：Web、CLI、桌面工具、腳本、API、Dashboard。
- 技術棧：語言、框架、套件、資料庫、部署環境。
- 模組邊界：UI、資料存取、業務邏輯、外部服務。
- 資料模型：主要實體、欄位、生命週期、保存策略。
- 介面契約：API、CLI 參數、設定檔、輸入輸出格式。
- 錯誤處理：驗證、重試、timeout、失敗通知。
- 觀測性：log、指標、debug 資訊、告警。
- 部署與執行：本機、伺服器、排程、容器或雲端服務。

### 產出

- `technical-design.md`
- 架構圖
- 資料流
- API / function / CLI contract
- 技術決策與取捨

## 4. 任務拆解

目標是把規格拆成小而明確的工作。

### 任務應包含

- 任務名稱
- 目的
- 涉及模組
- 完成條件
- 測試方式
- 依賴項
- 狀態

### 建議順序

1. 專案初始化與基本設定
2. 核心資料模型
3. 核心流程
4. UI / CLI / API 介面
5. 錯誤處理與邊界情境
6. 測試
7. 文件
8. 啟動、部署與維護腳本

## 5. 開發實作

目標是先做出最小可用版本，再逐步補齊品質。

### 原則

- 優先完成最短可用流程。
- 每次修改保持小範圍。
- 沿用既有專案風格。
- 設定值不要硬寫在程式中。
- 核心邏輯與介面層分離。
- 錯誤訊息要能協助定位問題。
- 重要流程要有簡短文件。

### AI 協作建議

- 先請 Codex 讀現有 repo，再要求修改。
- 大改動先產出 plan，再進入實作。
- 每次任務要求包含成功條件與驗證方式。
- 不確定處要求 Codex 標示假設與待確認。

## 6. 測試驗收

目標是確認工具真正可用，而不是只有程式能跑。

### 最低測試集合

- 正常流程
- 空資料
- 錯誤輸入
- 設定缺失
- 外部 API 失敗
- 權限不足
- 重複執行
- 主要 UI 尺寸或 CLI 使用方式

### 驗收問題

- 是否達成 `project-brief.md` 的成功條件？
- 是否符合 `system-spec.md` 的驗收條件？
- 是否有清楚啟動方式？
- 是否有錯誤排查方式？
- 是否能交給未來的自己維護？

## 7. 使用、維護與迭代

目標是讓工具在完成後仍能被理解、操作、修復與改版。

### 必要文件

- `README.md`
- 安裝方式
- 啟動方式
- 設定方式
- 常見錯誤
- 更新紀錄
- 後續需求清單

### 迭代流程

1. 收集使用問題。
2. 判斷是 bug、需求變更、資料問題還是操作問題。
3. 更新規格或 README。
4. 小範圍修改。
5. 測試。
6. 記錄版本變更。

## 8. 最小文件組

| 文件 | 目的 |
|---|---|
| `project-brief.md` | 目標、背景、使用者、成功條件 |
| `system-spec.md` | 功能、流程、驗收條件 |
| `technical-design.md` | 技術架構、資料流、介面 |
| `tasks.md` | 任務拆解與進度 |
| `README.md` | 實際使用與維護方式 |
| `acceptance-checklist.md` | 發布前驗收 |
